A forma recomendada de saber que uma transação foi paga é o [webhook](/api-reference/eventos). Para reconciliação ou como fallback, você pode consultar o status diretamente.

## Consultar uma transação

```http
GET /v1/transactions/{transactionId}
```

```bash
curl https://api.ephra.io/v1/transactions/clxabc123def456 \
  -H "Authorization: Bearer $TOKEN"
```

### Resposta

```json
{
  "success": true,
  "data": {
    "id": "clxabc123def456",
    "companyId": "clx...",
    "paymentMethod": "pix",
    "amount": 10000,
    "netAmount": 9850,
    "fees": 150,
    "status": "paid",
    "externalId": "...",
    "createdAt": "2026-06-03T12:00:00.000Z",
    "aprovedAt": "2026-06-03T12:01:30.000Z",
    "refuseReason": null,
    "customer": {
      "id": "...",
      "name": "Maria Silva",
      "document": "12345678909",
      "documentType": "cpf",
      "emails": [{ "email": "maria@email.com" }]
    },
    "Pix": { "key": "...", "endToEndId": "E...", "payerInfo": {} }
  }
}
```

<ResponseField name="data.status" type="string">
  Status atual da transação. Veja todos os valores em [Status e erros](/api-reference/status-e-erros).
</ResponseField>

<ResponseField name="data.amount" type="number">
  Valor bruto, em centavos.
</ResponseField>

<ResponseField name="data.netAmount" type="number">
  Valor líquido (após taxas), em centavos.
</ResponseField>

<ResponseField name="data.aprovedAt" type="string | null">
  Data de aprovação, ou `null` se ainda não foi paga.
</ResponseField>

## Listar transações

```http
GET /v1/transactions
```

Retorna uma lista paginada. Filtre por querystring:

<ParamField query="offset" type="string">
  Deslocamento para paginação. Padrão `0`.
</ParamField>

<ParamField query="status" type="string">
  Filtra por status (ex.: `paid`, `pending`, `refused`).
</ParamField>

<ParamField query="paymentMethod" type="string">
  `pix`, `credit_card` ou `boleto`.
</ParamField>

<ParamField query="transactionId" type="string">
  Filtra por ID da transação.
</ParamField>

<ParamField query="email" type="string">
  Filtra pelo e-mail do cliente.
</ParamField>

<ParamField query="phone" type="string">
  Filtra pelo telefone do cliente.
</ParamField>

<ParamField query="document" type="string">
  Filtra pelo CPF/CNPJ do cliente.
</ParamField>

<ParamField query="name" type="string">
  Filtra pelo nome do cliente.
</ParamField>

<ParamField query="initialDate" type="string">
  Data inicial do período (`YYYY-MM-DD`).
</ParamField>

<ParamField query="finalDate" type="string">
  Data final do período (`YYYY-MM-DD`).
</ParamField>

A resposta tem o formato `{ "success": true, "data": [ ... ], "totalRows": 0 }`.

<Tip>
  Evite polling agressivo. Para saber quando uma transação muda de status, prefira sempre o [webhook](/api-reference/webhooks).
</Tip>
