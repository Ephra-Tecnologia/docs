Quando uma transação muda de estado, a Ephra faz um `POST` para a URL que você cadastrar. É assim que você descobre, por exemplo, que um PIX foi pago — sem precisar ficar consultando.

## Cadastrar um webhook

```http
POST /v1/webhook
```

### Corpo da requisição

<ParamField body="url" type="string" required>
  URL HTTPS que receberá as notificações. É testada com um `POST` no momento do cadastro.
</ParamField>

<ParamField body="events" type="string[]" required>
  Lista de eventos que você quer receber. Ao menos um é obrigatório. Veja a lista em [Eventos e payload](/api-reference/eventos).
</ParamField>

<ParamField body="name" type="string">
  Nome para identificar o webhook.
</ParamField>

<ParamField body="isActive" type="boolean">
  Habilita ou desabilita a entrega. Padrão `true`.
</ParamField>

<ParamField body="productIds" type="string[]">
  Limita o webhook a produtos específicos. Se omitido, vale para todos.
</ParamField>

### Exemplo

```bash
curl -X POST https://api.ephra.io/v1/webhook \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "url": "https://seusite.com/webhooks/ephra",
    "name": "Produção",
    "events": ["transaction_created", "transaction_paid", "transaction_refunded"]
  }'
```

### Resposta

```json
{
  "success": true,
  "data": {
    "id": 42,
    "name": "Produção",
    "url": "https://seusite.com/webhooks/ephra",
    "isActive": true,
    "events": ["transaction_created", "transaction_paid", "transaction_refunded"],
    "signatureSecret": "a1b2c3...",
    "productIds": [],
    "createdAt": "2026-06-03T12:00:00.000Z"
  }
}
```

<Warning>
  Guarde o `signatureSecret`. Você precisa dele para validar a assinatura de cada webhook recebido. Veja [Validar a assinatura](/guias/webhooks#valide-a-assinatura).
</Warning>

## Alternativa: `postbackUrl` por transação

Se você precisa receber webhooks de apenas uma cobrança específica, informe o campo `postbackUrl` no corpo da cobrança PIX ou cartão. Esse endpoint recebe os eventos somente daquela transação, sem precisar cadastrar um webhook permanente.

## Gerenciar webhooks

| Método | Rota | Ação |
| --- | --- | --- |
| `GET` | `/v1/webhook` | Lista os webhooks da empresa (paginado). |
| `GET` | `/v1/webhook/{id}` | Detalha um webhook. |
| `DELETE` | `/v1/webhook/{id}` | Remove um webhook. |

### Supressões

Em cenários de alto volume, você pode suprimir parte das entregas de um evento usando as rotas `/v1/webhook/suppressions`. Uma supressão define, para um evento, quantas entregas suprimir a cada intervalo (anti-flood).
